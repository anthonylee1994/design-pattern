# 建造者模式 (Builder Pattern)

建造者模式是一種創建型設計模式，它可以讓你分步驟創建複雜對象。該模式允許你使用相同的創建代碼生成不同類型和形式的對象。

## 目的

- 將一個複雜對象的構建與它的表示分離，使得同樣的構建過程可以創建不同的表示。
- 當需要創建的對象具有複雜的內部結構時，使用建造者模式可以簡化創建過程。
- 允許對象通過多個步驟來創建，並且可以控制每個步驟。

## 實現示例

在這個示例中，我們使用建造者模式來創建一個漢堡（Burger）對象。漢堡可以有不同的配料組合：

```java
Burger burger = new Burger.Builder(14)
        .addCheese()
        .addLettuce()
        .addTomato()
        .build();
```

### 類的結構

- `Burger`：最終產品類

  - 包含漢堡的所有屬性（大小、芝士、意大利辣香腸、生菜、番茄）
  - 使用內部靜態類 `Builder` 來構建漢堡對象

- `Builder`：建造者類
  - 提供流式接口來添加不同的配料
  - 通過 `build()` 方法完成漢堡的創建

### 優點

1. 可以逐步創建對象，並且可以靈活控制創建過程
2. 同樣的創建代碼可以創建不同的產品
3. 隔離了複雜對象的創建和使用
4. 更好的控制創建過程
5. 產品的內部表示可以變化，而不會影響客戶端代碼

### 使用場景

- 當對象需要通過多個步驟來創建時
- 當對象的創建過程應該獨立於組成對象的部件及其裝配方式時
- 當構造過程需要以不同的方式構造對象時
